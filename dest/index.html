ebbieについて

ebbie は小規模開発での利用を想定して作ったsass/gulpの集まりです。 基本的な指針はありますが、利用する方のプロジェクト毎にカスタマイズして利用する事を想定しています。

ライブラリ

MixinライブラリとしてBourbonを利用しています。 グリッドフレームワークとしてNEATも使えるようにしていますが、こちらは任意でご利用下さい。
ドキュメントとしてSassDocを利用しています。

主なGulpタスク

  • $ gulp - SASS(SCSS), HTML, (gulpfile.jsに定義している)JavaScript の変更を検知(SourceMapsの付与, Autoprefixerの適用, JavaScriptの連結)してブラウザに反映
  • $ gulp stylegd - SASS(SCSS) の変更を検知してブラウザに反映
  • $ gulp build - SASS(SCSS) コンパイル, js ファイルの連結, スプライト画像の作成, cssのプロパティ並び替え、MediaQueryを纏める
  • $ gulp release - SASS(SCSS) コンパイル, js ファイルの連結・圧縮, スプライト画像の作成・画像圧縮, cssのプロパティ並び替え、MediaQueryを纏める
  • $ gulp js-concat - js ファイルの連結
  • $ gulp js-uglify - js ファイルの圧縮
  • $ gulp sprite - スプライト画像の作成
  • $ gulp imagemin - 画像ファイルの圧縮

gulp タスク実行時のサーバー

  • http://localhost:3000 - 開発環境
  • http://localhost:3000/sassdoc - SassDoc環境

ディレクトリ構成

  • html - dest以下で管理
  • その他のファイル(SASS(SCSS)/js/img) - src以下で管理し、各gulpタスクによってdestへ出力
+ dest/                         # 出力ディレクトリ
  | + css/                      # css
  |   |-- main.css              # コンパイルしたcssファイル
  |
  | + img/                      # 圧縮した画像ファイル
  |
  | + js/                       # jsファイル
  |   |-- main.js               # 結合したjsファイル
  |
  | + sassdoc/                  # SassDocドキュメント
  |
  + htmlファイル                 # htmlファイル
  |
+ src/
  |
  | + img/                      # 画像ファイル
  |   + sprite/                 # スプライト用画像ファイル(pngファイル)
  |
  | + js/                       # jsファイル
  |
  | + sass/                     # SASS(SCSS)ファイル
  |   + components/             # 共通コンポーネント
  |     |-- _import.scss        # importファイル
  |     |-- _xxx.scss           # コンポーネントファイル
  |     | ...
  |
  |   + foundation/             # 変数やhelperを管理するコンポーネント
  |     |-- _import.scss        # importファイル
  |     |-- _base.scss          # サイト全体のベースとなるファイル
  |
  |   + helpers/                # helperを管理するコンポーネント
  |     |-- _import.scss        # importファイル
  |     |-- _functions.scss     # functionsを定義
  |     |-- _mixins.scss        # mixinを定義
  |     |-- _placeholders.scss  # placeholdersを定義
  |
  |   + layout/                 # ヘッダー、サイドバー、フッターなどのレイアウトを管理するコンポーネント
  |     |-- _import.scss        # importファイル
  |     |-- _header.scss        # ヘッダーレイアウト
  |     |-- _contents.scss      # コンテンツレイアウト
  |     |-- _footer.scss        # フッターレイアウト
  |     | ...
  |
  |   + utils/                  # 汎用クラスコンポーネント
  |     |-- _import.scss        # importファイル
  |     |-- _utility.scss       # 汎用クラスを定義
  |
  |   + _pallette.scss          # 全体の色を定義
  |   + _sprite.scss            # gulpタスクで生成されるSpriteを扱うファイル（自動生成）
  |   + _variables.scss         # 全体の変数を定義
  |   + main.scss               # 全てのimport ファイル
  |
+ .csscomb.json                 # csscomb の設定ファイル
+ .gitignore                    # gitignore
+ gulpfile.js                   # Gulp の定義とタスク
+ LICENSE                       # ライセンスファイル
+ package.json                  # Gulp のメタ情報とプラグイン情報
+ README.md                     # READMEファイル

Prefixと役割

u-* - utility
paddingやmargin等の汎用クラスを作成して利用します。但し、肥大化するとメンテナンス性が落ちます
l-* - layout
ヘッダー、サイドバー、フッターなどのレイアウトを管理します。粒度の大きいコンポーネントなので、作成しなくても構いません
is-* - state
JavaScriptなどで状態を変化させる際に利用します
